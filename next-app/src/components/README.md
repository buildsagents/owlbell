# Components

The previous UI was archived. Rebuild new components here intentionally.

Suggested structure:

- `ui/` - small reusable primitives such as buttons, inputs, badges, dialogs.
- `layout/` - header, footer, navigation, page shells.
- `sections/` - marketing/site sections for the agency site.
- `features/` - larger product areas such as audit form, demo, pricing, reports.

Do not add large one-off pages full of duplicated styling. Build small, named pieces.
